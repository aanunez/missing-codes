How to use the @MissingCode Action Tag

Purpose
This new action tag allows you to add buttons below a text or notes entry box with options such as "Missing" or "Don't know". These buttons, when selected, will disable entry on the box and populate it with a predetermined code. Available options include ...
  • Not Applicable
  • Prefer not to answer
  • Refused
  • Declined
  • Don't Know
  • Missing
Prerequisites
Using this action tag requires no special knowledge, changes to your project settings, or intervention by administrators.
Using the action tag
  • Create a text or notes field.
  • In the Action Tags / Field Annotation
    Add the @MISSINGCODE=SYMBOL annotation (no quotes around the symbol)
  • The following symbols can be used with the tag
    Symbol Coded Button Text
    NA -6 Not Applicable
    PF -7 Prefer not to answer
    RF -7 Refused
    DC -7 Declined
    DK -8 Don't Know
    MS -9 Missing
  • Examples
    Action Tags / Field Annotation
    @MISSINGCODE=DK,PF
                            //Insert picture here
    @MISSINGCODE=NA
                            //Insert picture here
    @MISSINGCODE=DC,NA @CHARLIMIT=4
                            //Insert picture here
Known issues
  • The buttons created by this action tag do not display on the "preview instrument" page
  • The action tag does not respect the "@READONLY" action tag. As such the two tags should not be used in combination with one another.
  • When used with the "@CHARLIMIT" action tag, "@MISSINGCODE" must be first. This is recognized as a bug and is being investigated.
